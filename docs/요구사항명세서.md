## 1. 개요

### 1.1 목적

본 문서는 벤포드 법칙(Benford's Law)을 기반으로 수치 데이터의 통계적 이상 여부를 탐지하는 웹 애플리케이션의 기능적, 비기능적 요구사항을 정의합니다.

### 1.2 범위

이 시스템은 사용자가 제공한 수치 데이터에 대해 앞자리 숫자 분포를 분석하고, 카이제곱 검정을 통해 벤포드 법칙 부합 여부를 판정합니다. 결과는 통계 수치와 시각화 차트 형태로 제공됩니다.

### 1.3 용어 정의

| 용어 | 정의 |
|:----:|:----:|
| 벤포드 법칙 | 자연 발생 수치 데이터에서 앞자리 숫자가 로그 분포를 따른다는 통계 법칙 |
| 앞자리 숫자 | 숫자에서 가장 왼쪽에 위치한 0이 아닌 유효숫자 |
| 카이제곱 검정 | 관측 빈도와 기대 빈도의 적합도를 검정하는 통계 방법 |
| p-값 | 귀무가설(벤포드 법칙을 따름)이 참일 때 관측값 이상의 결과가 나올 확률 |

## 3. 기능 요구사항

### 3.1 데이터 입력 (FR-01)

#### FR-01-1: 직접 입력
- 사용자는 쉼표(,)로 구분된 숫자를 텍스트 박스에 직접 입력할 수 있어야 한다.
- 입력값에 공백이 포함되어도 정상적으로 처리되어야 한다.
- 유효하지 않은 입력(문자, 특수문자)에 대해 사용자에게 오류 메시지를 표시해야 한다.

#### FR-01-2: 샘플 데이터 제공
- 시스템은 최소 3종의 샘플 데이터를 제공해야 한다:
  - 정상 데이터: 벤포드 법칙을 따르는 로그정규분포 데이터 (n=500)
  - 조작 데이터: 특정 앞자리 숫자(4~7)에 집중된 데이터 (n=500)
  - 균등 난수: 균등 분포 난수 (n=500)

#### FR-01-3: CSV 파일 업로드
- 사용자는 .csv 형식의 파일을 업로드하여 분석을 수행할 수 있어야 한다.
- CSV 파일의 모든 수치 열을 자동으로 읽어 분석에 사용해야 한다.
- 비수치 열은 자동으로 무시해야 한다.

### 3.2 앞자리 숫자 추출 (FR-02)

#### FR-02-1: 추출 로직
- 양수, 음수(절댓값 사용), 소수(예: 0.0071 → 7) 모두 처리할 수 있어야 한다.
- 0은 유효 앞자리가 없으므로 분석에서 제외해야 한다.
- 과학적 표기법으로 표현된 숫자도 정상 처리되어야 한다.

#### FR-02-2: 유효 데이터 검증
- None, NaN, 무한대(inf) 값은 자동으로 제외해야 한다.
- 유효 데이터 수(n)를 사용자에게 표시해야 한다.

### 3.3 벤포드 법칙 분석 (FR-03)

#### FR-03-1: 기대값 계산
- 벤포드 법칙 공식 `P(d) = log10(1 + 1/d)`을 사용하여 각 앞자리 숫자(1~9)의 기대 확률을 계산해야 한다.

#### FR-03-2: 카이제곱 검정
- 관측 빈도(O_d)와 기대 빈도(E_d = n * P(d))를 사용하여 카이제곱 검정을 수행해야 한다.
- 퍼센트가 아닌 빈도(count)를 사용해야 한다.
- 자유도는 8 (9개 범주 - 1)이어야 한다.

#### FR-03-3: 판정
- p-값이 0.05 초과이면 "정상 (벤포드 법칙 부합)"으로 판정해야 한다.
- p-값이 0.05 이하이면 "조작 의심 (벤포드 법칙 위반)"으로 판정해야 한다.

### 3.4 결과 표시 (FR-04)

#### FR-04-1: 요약 정보
- 판정 결과를 녹색(정상) 또는 빨간색(조작 의심) 배너로 표시해야 한다.
- 카이제곱 통계량과 p-값을 소수점 4자리까지 표시해야 한다.
- 분석에 사용된 유효 데이터 수(n)를 표시해야 한다.

#### FR-04-2: 시각화
- 앞자리 숫자 1~9에 대한 실제 분포(%)와 벤포드 예상 분포(%)를 그룹형 막대 차트로 나란히 표시해야 한다.
- 차트는 인터랙티브하게 동작해야 하며 (hover, zoom 등) 범례를 포함해야 한다.

#### FR-04-3: 상세 비교표
- 앞자리 숫자 1~9 각각에 대해 실제 빈도, 실제 비율(%), 벤포드 예상 비율(%)을 표 형식으로 제공해야 한다.

#### FR-04-4: 수학 원리 설명
- 벤포드 법칙 공식과 카이제곱 검정 공식을 LaTeX 수식으로 표시해야 한다.
- 접기/펼치기 방식의 expander로 제공해야 한다.

#### FR-04-5: 활용 사례 안내
- 벤포드 법칙의 실제 활용 사례(회계 감사, 선거 검증 등)를 접기/펼치기 방식으로 제공해야 한다.
- 법칙의 한계 및 주의사항을 함께 안내해야 한다.

## 4. 비기능 요구사항

### 4.1 성능 (NFR-01)
- 500개 데이터 분석은 1초 이내에 완료되어야 한다.
- 10,000개 데이터 분석은 5초 이내에 완료되어야 한다.
- CSV 파일 업로드 및 파싱은 파일 크기 10MB 이내에서 10초 이내 완료되어야 한다.

### 4.2 사용성 (NFR-02)
- 전체 UI는 한국어로 제공되어야 한다.
- 처음 사용하는 사용자도 별도의 교육 없이 3분 이내에 분석을 완료할 수 있어야 한다.
- 모바일 브라우저에서도 기본 기능이 동작해야 한다 (반응형 레이아웃).

### 4.3 신뢰성 (NFR-03)
- 잘못된 입력값(문자열, 빈 값 등)이 있어도 애플리케이션이 비정상 종료되지 않아야 한다.
- CSV 파일에 수치 외 컬럼이 있어도 정상적으로 처리되어야 한다.

### 4.4 유지보수성 (NFR-04)
- 핵심 분석 로직(`benford/core.py`)과 UI(`app.py`)가 명확히 분리되어야 한다.
- 데이터 로딩 로직은 별도 모듈(`benford/data_loader.py`)에 위치해야 한다.
- 코드는 Python PEP 8 스타일 가이드를 준수해야 한다.

### 4.5 이식성 (NFR-05)
- Windows, macOS, Linux 환경에서 동일하게 동작해야 한다.
- `requirements.txt`를 통한 의존성 설치만으로 실행 가능해야 한다.

## 5. 데이터 요구사항

### 5.1 입력 데이터 형식

| 입력 방식 | 형식 | 제약사항 |
|:---------:|:----:|:--------:|
| 직접 입력 | 쉼표 구분 숫자 문자열 | 최소 10개 권장 |
| CSV 업로드 | UTF-8 인코딩 CSV | 최대 10MB |
| 샘플 데이터 | 내장 생성 | 고정 시드(seed=42) 사용 |

### 5.2 출력 데이터

| 출력 항목 | 형식 | 설명 |
|:---------:|:----:|:----:|
| 판정 결과 | 문자열 | "정상" 또는 "조작 의심" |
| 카이제곱 통계량 | 부동소수점 (소수점 4자리) | χ² 값 |
| p-값 | 부동소수점 (소수점 4자리) | 유의확률 |
| 앞자리 분포 | 딕셔너리 (d: count) | 1~9 각각의 관측 빈도 |
| 비율 데이터 | 딕셔너리 (d: float) | 0~1 사이의 소수 |

## 6. 제약사항 및 가정

### 6.1 제약사항

- 최소 유효 데이터 수는 분석의 신뢰성을 위해 30개 이상을 권장한다.
- 특정 구간으로 제한된 데이터(예: 나이 0~100, 시급 등)는 벤포드 법칙이 적용되지 않을 수 있다.
- 이 도구는 법적 증거로 사용할 수 없으며, 추가 조사의 참고 자료로만 활용해야 한다.

### 6.2 가정

- 사용자는 통계에 대한 기초 지식을 보유하고 있다고 가정한다.
- 입력 데이터는 자연 발생 데이터 또는 의도적으로 조작된 데이터 중 하나라고 가정한다.
