# 벤포드 검증기 (Benford Detector)

> 숫자 속 거짓을 찾는 데이터 검증 도구
> **확률과 통계 × 미디어 리터러시 융합 프로젝트**

AI와 가짜뉴스의 시대, 우리는 매일 수많은 통계·조회수·후원금 숫자를 마주합니다.
이 도구는 **벤포드 법칙**이라는 수학 원리로 그 숫자들이 자연스러운지,
아니면 누군가 지어낸 것인지 검증합니다.

## 한눈에 보기

자연스럽게 발생한 데이터의 **첫 번째 유효숫자**는 1~9가 고르게 나오지 않습니다.
1이 약 30%로 압도적으로 많고, 9로 갈수록 줄어듭니다. 이것이 **벤포드 법칙**입니다.

```
P(d) = log₁₀(1 + 1/d)
```

| 첫자리 | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 | 9 |
|---|---|---|---|---|---|---|---|---|---|
| 기대 비율 | 30.1% | 17.6% | 12.5% | 9.7% | 7.9% | 6.7% | 5.8% | 5.1% | 4.6% |

사람이 숫자를 조작하면 이 분포가 깨집니다. 이 도구는 **카이제곱 적합도 검정**으로
관측된 분포가 벤포드 법칙에서 통계적으로 유의미하게 벗어났는지 판정합니다.

## 사용법

### 웹앱 (가장 쉬움)
1. `index.html`을 브라우저로 열거나, 아래 배포 링크로 접속
2. 검증할 숫자를 붙여넣기 (쉼표·줄바꿈·공백으로 구분)
3. **분석하기** 클릭 → 그래프와 판정 결과 확인
4. 예시 버튼(자연/조작/균등)으로 차이를 즉시 비교

### 파이썬 엔진
```bash
python3 benford.py
```
자연 데이터와 조작 데이터를 비교하는 데모가 실행됩니다.
직접 분석하려면:
```python
from benford import analyze
result = analyze([145, 1700, 23, 410, 56, 678, 104, 220])
print(result["verdict"], result["chi_square"])
```

## 작동 원리 (수학)

**1. 첫자리 추출** — 숫자를 1.0 이상 10.0 미만이 될 때까지 10으로 곱하거나 나눠
정수부를 취합니다. (예: 4823 → 4, 0.0071 → 7)

**2. 분포 비교** — 입력 데이터에서 1~9 각 첫자리의 실제 비율을 세고
벤포드 기대 비율과 나란히 그립니다.

**3. 카이제곱 적합도 검정**
```
χ² = Σ (관측빈도 - 기대빈도)² / 기대빈도
```
자유도 8에서 임계값은 **15.507** (유의수준 0.05)입니다.
χ²가 이 값을 넘으면 "벤포드 분포에서 유의미하게 벗어남 = 조작 의심"으로 판정합니다.

## ⚠️ 한계와 책임 있는 사용

- 이 도구의 결과는 **"추가 검증이 필요하다는 신호"**일 뿐, 조작의 직접적 증거가 아닙니다.
- 표본이 작으면(권장 30개 이상) 정상 데이터도 우연히 벗어날 수 있습니다.
- 데이터 성격에 따라 정상인데도 벤포드를 따르지 않는 경우가 있습니다.
  (예: 가격처럼 상한이 정해진 값, 학번·전화번호처럼 할당된 번호, 한 자릿수에 몰린 값)
- 따라서 어떤 데이터를 함부로 "조작"이라 단정하지 않습니다.
