# 🔢 벤포드 법칙 데이터 검증기 (Benford's Law Detector)

자연스럽게 발생한 데이터의 **맨 앞자리(첫 유효숫자)** 분포를 분석해, 어떤 데이터가
**벤포드 법칙을 따르는지(정상)** 아니면 **벗어나는지(조작 의심 신호)** 를 통계적으로
판정하는 파이썬 프로그램과 인터랙티브 대시보드입니다.

> 교과 연계: **확률과 통계**(확률분포·가설검정)와 **로그/지수**(상용로그) 개념을
> 실제 데이터에 적용한 프로젝트입니다.

---

## 1. 벤포드 법칙이란?

인구, 강의 길이, 주가, GDP, 후원금처럼 여러 자릿수에 걸쳐 자연스럽게 발생한 데이터의
맨 앞자리를 모아 보면, 1~9가 고르게(각 11.1%) 나오지 않습니다. **1이 약 30%로 가장
많고, 9로 갈수록 적어집니다.**

첫자리 `d`가 나타날 확률은 다음 공식을 따릅니다.

```
P(d) = log10(1 + 1/d),   d = 1, 2, ..., 9
```

| 앞자리 d |   1   |   2   |   3   |  4   |  5   |  6   |  7   |  8   |  9   |
|:-------:|:-----:|:-----:|:-----:|:----:|:----:|:----:|:----:|:----:|:----:|
| 예상 비율 | 30.1% | 17.6% | 12.5% | 9.7% | 7.9% | 6.7% | 5.8% | 5.1% | 4.6% |

**왜 이렇게 되나?** 데이터가 로그 스케일에서 고르게 퍼지기 때문입니다. 1에서 2로 가는
"거리"가 로그상 가장 길어서, 1로 시작하는 구간에 가장 오래 머무릅니다.

**왜 조작 탐지에 쓰이나?** 사람이 숫자를 지어내면 무의식적으로 앞자리를 고르게 쓰거나
특정 숫자를 선호해 이 분포가 깨집니다. 실제로 회계 부정 감사, 선거·통계 검증 등에
1차 선별 도구로 쓰입니다.

## 2. 프로그램이 하는 일

```
숫자 데이터 입력 → 첫자리 추출 → 실제 분포 계산
  → 벤포드 예상 분포와 비교(그래프) → 카이제곱 적합도 검정
  → "정상 / 조작 의심" 판정 출력
```

확률과 통계 개념이 두 번 들어갑니다.
1. **벤포드 분포** (확률분포)
2. **카이제곱 적합도 검정** (가설검정)

```
chi^2 = Σ (관측빈도_d - 기대빈도_d)^2 / 기대빈도_d
```

자유도 8에서 이 값이 임계값(α=0.05일 때 ≈ 15.51)보다 크면 "유의미하게 다름 =
추가 검증이 필요한 신호"로 판정합니다.

## 3. 설치 및 실행

### 설치
```bash
git clone <이 저장소 주소>
cd project_f
pip install -r requirements.txt
```

### (A) 인터랙티브 대시보드 — 사람들이 체험하는 화면
```bash
streamlit run app.py
```
브라우저가 열리면 왼쪽에서 데이터를 고르거나 직접 입력/CSV 업로드해 결과를 봅니다.

### (B) 명령줄(CLI)로 분석
```bash
# 내장 예시 데이터 목록 보기
python analyze.py --list

# 예시 데이터 분석
python analyze.py --sample "한국 도시 인구"

# 내 CSV 파일의 특정 열 분석
python analyze.py --csv data/sample_numbers.csv --column 인구

# 한 줄에 숫자 하나씩 든 텍스트 파일 분석
python analyze.py --file my_numbers.txt
```

### (C) 코드로 직접 사용
```python
from benford import analyze

result = analyze([4823, 1290, 57, 0.0071, 980])
print(result.summary())
print("정상 여부:", result.follows_benford)
```

## 7. 윤리적 주의

벤포드 법칙은 **'신호'일 뿐 증거가 아닙니다.** 분포가 벗어났다고 해서 곧바로 조작이라고
단정할 수 없으며, 더 들여다볼 대상을 찾는 **1차 선별 도구**로만 사용해야 합니다.
실제 데이터를 분석할 때는 "추가 검증이 필요한 신호" 정도로 신중하게 표현하세요.
